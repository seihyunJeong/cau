### 1-4. 전문 용어 → 부모 언어 매핑

앱 화면에는 **부모 언어를 기본 표시**하되, 옆에 **ⓘ 아이콘**을 배치하여 탭하면 전문 용어명과 설명이 Tooltip으로 나타난다. 이를 통해:
- 초보맘은 부모 언어만 보고 바로 이해
- 궁금한 부모는 탭해서 전문 용어 학습
- 소아과 방문 시 "모로반사"라는 용어를 알고 소통 가능

**화면 표시 예시:**

```
"깜짝 놀라며 팔을 벌리는 반응 ⓘ"     ← 기본 상태

탭 시:
  ┌─────────────────────────────┐
  │  모로반사 (Moro Reflex)      │
  │                             │
  │  갑작스러운 소리나 움직임에    │
  │  놀라 양팔을 벌렸다가 모으는   │
  │  자연스러운 반응이에요.       │
  │  보통 3~4개월에 자연스럽게    │
  │  사라져요.                   │
  └─────────────────────────────┘
```

**용어 매핑 테이블:**

| 전문 용어 (Tooltip 제목) | 앱 화면 표현 (기본 텍스트) | Tooltip 설명 |
|---|---|---|
| 모로반사 (Moro Reflex) | 깜짝 놀라며 팔을 벌리는 반응 ⓘ | 갑작스러운 소리나 움직임에 놀라 양팔을 벌렸다가 모으는 자연스러운 반응이에요. 보통 3~4개월에 자연스럽게 사라져요. |
| 탐색반사 (Rooting Reflex) | 입 주변을 건드리면 고개를 돌리는 반응 ⓘ | 볼이나 입 주변을 건드리면 그쪽으로 고개를 돌리는 반응이에요. 수유와 연결되는 중요한 반응이에요. |
| 파악반사 (Palmar Grasp) | 손바닥을 건드리면 꽉 쥐는 반응 ⓘ | 손바닥에 무언가 닿으면 자동으로 꽉 쥐는 반응이에요. 나중에 물건 잡기의 기초가 돼요. |
| 각성 (Arousal) | 깨어 있는 시간 ⓘ | 아기가 잠에서 깨어 주변을 인식하는 상태예요. 신생아는 하루 중 깨어 있는 시간이 매우 짧아요. |
| 공동조절 (Co-regulation) | 부모가 안아주며 함께 안정되는 것 ⓘ | 아기가 스스로 진정하기 어려울 때, 부모의 안기·목소리·리듬으로 함께 안정되는 과정이에요. |
| 과자극 (Overstimulation) | 자극이 너무 많아 지친 상태 ⓘ | 빛, 소리, 움직임 등 자극이 아기가 처리할 수 있는 양을 넘어선 상태예요. 울음이나 고개 돌림으로 나타나요. |
| 굴곡 자세 (Flexion Posture) | 팔다리를 몸 쪽으로 모은 자세 ⓘ | 자궁 안에서 익숙했던 웅크린 자세예요. 신생아가 편안할 때 자연스럽게 이 자세를 취해요. |
| 신경계 통합 (Neural Integration) | 아기의 뇌와 몸이 안정을 배우는 과정 ⓘ | 아기의 뇌가 감각 정보를 받아들이고 처리하는 방법을 익혀가는 과정이에요. |
| consolability | 달래주면 진정되는 정도 ⓘ | 울거나 불편해할 때 보호자가 안아주면 얼마나 빨리 안정되는지를 나타내요. |

**InfoTerm 재사용 위젯 스펙:**

```dart
/// 전문 용어를 부모 언어로 표시하고, 탭하면 Tooltip으로 전문 용어 설명을 보여주는 위젯.
/// 앱 전체에서 전문 용어가 등장하는 모든 곳에 이 위젯을 사용한다.
///
/// 패키지: just_the_tooltip ^0.0.12 (탭 트리거 + 커스텀 콘텐츠 지원)

class InfoTerm extends StatelessWidget {
  final String parentLabel;     // 부모 언어 ("깜짝 놀라며 팔을 벌리는 반응")
  final String termName;        // 전문 용어명 ("모로반사 (Moro Reflex)")
  final String description;     // 설명 ("갑작스러운 소리나 움직임에...")

  const InfoTerm({
    required this.parentLabel,
    required this.termName,
    required this.description,
  });

  @override
  Widget build(BuildContext context) {
    return JustTheTooltip(
      triggerMode: TooltipTriggerMode.tap,
      backgroundColor: Color(0xFFFFF8F0),  // 크림/아이보리 배경
      borderRadius: BorderRadius.circular(12),
      tailLength: 8,
      tailBaseWidth: 12,
      content: Padding(
        padding: EdgeInsets.all(12),
        child: Column(
          mainAxisSize: MainAxisSize.min,
          crossAxisAlignment: CrossAxisAlignment.start,
          children: [
            Text(
              termName,
              style: TextStyle(
                fontWeight: FontWeight.bold,
                fontSize: 14,
                color: Color(0xFF3D3027),  // 다크 브라운
              ),
            ),
            SizedBox(height: 6),
            Text(
              description,
              style: TextStyle(
                fontSize: 13,
                color: Color(0xFF8C7B6B),  // 웜 그레이
                height: 1.5,
              ),
            ),
          ],
        ),
      ),
      child: Row(
        mainAxisSize: MainAxisSize.min,
        children: [
          Flexible(
            child: Text(
              parentLabel,
              style: TextStyle(fontSize: 15, color: Color(0xFF3D3027)),
            ),
          ),
          SizedBox(width: 4),
          Icon(
            Icons.info_outline,
            size: 16,
            color: Color(0xFF8C7B6B),
          ),
        ],
      ),
    );
  }
}
```

**사용 예시:**

```dart
InfoTerm(
  parentLabel: '깜짝 놀라며 팔을 벌리는 반응',
  termName: '모로반사 (Moro Reflex)',
  description: '갑작스러운 소리나 움직임에 놀라 양팔을 벌렸다가 모으는 자연스러운 반응이에요. 보통 3~4개월에 자연스럽게 사라져요.',
)
```

**사용되는 화면:**
- 활동 상세 (연결 반응 표시)
- 관찰 기록 폼 (2단계 자연 반응 항목)
- 발달 체크 탭 (발달 포인트 요약)

```dart
  final int? id;
  final int babyId;
  final int weekNumber;
  final DateTime date;
  final String signs;           // JSON: {"danger_0": true, "danger_1": false, ...}
  final String? memo;
  final bool hasAnySign;        // 1개 이상 체크 여부 (빠른 조회용)
  final DateTime createdAt;

  const DangerSignRecord({
    this.id,
    required this.babyId,
    required this.weekNumber,
    required this.date,
    required this.signs,
    this.memo,
    this.hasAnySign = false,
    required this.createdAt,
  });

  Map<String, dynamic> toMap() => {
    'id': id,
    'baby_id': babyId,
    'week_number': weekNumber,
    'date': date.toIso8601String(),
    'signs': signs,
    'memo': memo,
    'has_any_sign': hasAnySign ? 1 : 0,
    'created_at': createdAt.toIso8601String(),
  };

  factory DangerSignRecord.fromMap(Map<String, dynamic> map) => DangerSignRecord(
    id: map['id'] as int?,
    babyId: map['baby_id'] as int,
    weekNumber: map['week_number'] as int,
    date: DateTime.parse(map['date'] as String),
    signs: map['signs'] as String,
    memo: map['memo'] as String?,
    hasAnySign: (map['has_any_sign'] as int) == 1,
    createdAt: DateTime.parse(map['created_at'] as String),
  );
}
```

```dart
/// domain_scores JSON을 앱 레이어에서 파싱하여 영역별 평균을 계산한다.
Future<Map<String, double>> getDomainAverages(int babyId, {int limit = 5}) async {
  final db = await DatabaseHelper.instance.database;
  final maps = await db.rawQuery('''
    SELECT domain_scores FROM checklist_records
    WHERE baby_id = ? ORDER BY date DESC LIMIT ?
  ''', [babyId, limit]);

  final Map<String, List<int>> domainValues = {};
  for (final map in maps) {
    final scores = jsonDecode(map['domain_scores'] as String) as Map<String, dynamic>;
    scores.forEach((domain, score) {
      domainValues.putIfAbsent(domain, () => []).add(score as int);
    });
  }

  return domainValues.map(
    (domain, values) => MapEntry(domain, values.average),
  );
}
```

#### 화면 D: 알림 권한 요청 화면 (NotificationPermissionScreen)

**기능:**
- 알림의 **가치를 먼저 설명**한 뒤 OS 권한 팝업 요청
- 허용 시: 기본 알림 스케줄 등록 (데일리 미션, 주차 전환, 마일스톤) 후 홈으로 이동
- 거부 시: 알림 없이 앱 사용 가능 — 부정적 메시지 없이 바로 홈으로 이동
- 거부해도 마이탭 > 알림 설정에서 언제든 다시 켤 수 있음
- 이 화면 완료 후 `isOnboardingComplete = true` 설정

**권한 처리 로직:**
```dart
// Android 13+ (API 33): POST_NOTIFICATIONS 런타임 권한
// Android 12+ (API 31): SCHEDULE_EXACT_ALARM 권한 (정확한 시간 예약)
// iOS: 항상 런타임 권한 필요 (최초 1회 시스템 팝업)

import 'package:permission_handler/permission_handler.dart';

Future<bool> requestNotificationPermission() async {
  if (Platform.isAndroid) {
    final status = await Permission.notification.request();
    if (status.isGranted) {
      // Android 12+: 정확한 알람 권한도 요청
      await Permission.scheduleExactAlarm.request();
    }
    return status.isGranted;
  }

  if (Platform.isIOS) {
    final status = await Permission.notification.request();
    return status.isGranted;
  }

  return false;
}
```

### 5-4. 탭 2: 기록 (RecordMainScreen) — "우리 아이 하루 기록"

**기능:**
- 날짜 선택 (기본: 오늘, 좌우 스와이프로 날짜 이동)
- 수유 횟수 카운터 (+1/−1 버튼, 수정 가능)
- 배변 횟수 카운터 (+1/−1 버튼, 수정 가능)
- 수면 시간 입력 (시간 피커 또는 슬라이더) — 수면은 기록 탭에서만 상세 입력
- 성장 기록 (체중/키/머리둘레): 기본 접힘 상태의 ExpansionTile로 구성
  - 기본 상태: "성장 기록 추가하기 +" 텍스트로 접혀 있음 (빈 필드 노출 방지 → 죄책감 없음)
  - 오늘 데이터 존재 시: 마지막 측정값을 컴팩트 카드로 표시
  - 펼치면 체중/키/머리둘레 입력 필드 노출
- 부모 메모 (자유 텍스트, 선택)
- 입력 시 즉시 안심/주의 메시지 표시 (숫자보다 말로)
- 아무 항목 1개만 입력해도 "오늘 기록 완료" 인정 (입력 최소화)

  - 수면 시간, 성장 데이터(체중/키/머리둘레) 입력 시: 해당 필드 옆에 인라인 "✓ 저장됨" 텍스트가 표시되었다가 2초 후 페이드아웃
  - 부모 메모 입력 시에도 동일하게 "✓ 저장됨" 인라인 표시

#### 화면 3-1: 활동 상세 (ActivityDetailScreen)

**기능:**
- 2레이어 구조: "하기 모드"(Layer 1)와 "더 알아보기"(Layer 2)로 분리
- **Layer 1 — 하기 모드 (화면 진입 시 보이는 영역):**
  - 활동명 + 한 줄 설명
  - 단계별 가이드 (steps 1~5)
- **Layer 2 — 더 알아보기 (접힌 상태, 펼침 가능):**
  - 관찰 포인트
  - 왜 하나요? (근거 설명)
  - 기대 효과
  - 이런 때는 잠깐 멈춰주세요 (기존 "주의사항" → 부모 언어로 변경)

  - "전문가가 설계한 활동이에요" (기존 "NBAS 기반 활동 설계" → 부모 언어로 변경)

#### 화면 4-1: 발달 체크리스트 (ChecklistScreen)

**기능:**
- 6개 영역 x 3문항 = 18문항

## 6. 점수/계산 로직 (Dart 코드)

### 6-1. 주차 계산기 (WeekCalculator)

```dart
/// 생년월일로부터 현재 주차를 계산한다.
/// 주차 정의:
///   0-1주: 생후 0~13일 (0~1주)
///   2-3주: 생후 14~27일 (2~3주)
///   4-5주: 생후 28~41일 (4~5주)
///   ...이후 2주 단위로 증가
///   4개월 이후: 월 단위로 전환
class WeekCalculator {
  /// 생년월일로부터 현재 주차 구간을 반환한다.
  /// 반환값: 주차 구간 문자열 (예: "0-1주", "2-3주")
  static String calculateWeekLabel(DateTime birthDate) {
    final now = DateTime.now();
    final days = now.difference(birthDate).inDays;

    if (days < 0) return "0-1주";

    if (days <= 13) return "0-1주";
    if (days <= 27) return "2-3주";

    // 4주 이후: 2주 단위로 계산
    final weeks = days ~/ 7;
    if (weeks < 8) {
      final startWeek = ((weeks ~/ 2) * 2);
      return "$startWeek-${startWeek + 1}주";
    }

    // 2개월 이후: 월 단위
    final months = days ~/ 30;
    if (months <= 60) {
      return "$months개월";
    }

    return "60개월+";
  }

  /// 생년월일로부터 현재 주차 번호를 반환한다 (콘텐츠 매핑용).
  /// 0 = 0-1주, 1 = 2-3주, 2 = 4-5주, ...
  static int calculateWeekIndex(DateTime birthDate) {
    final now = DateTime.now();
    final days = now.difference(birthDate).inDays;

    if (days < 0) return 0;
    if (days <= 13) return 0;   // 0-1주
    if (days <= 27) return 1;   // 2-3주

    final weeks = days ~/ 7;
    return weeks ~/ 2;
  }

  /// 태어난 지 며칠인지 반환한다.
  static int daysSinceBirth(DateTime birthDate) {
    return DateTime.now().difference(birthDate).inDays;
  }
}
```

### 6-2. 점수 계산기 (ScoreCalculator)

```dart
/// 발달 체크리스트 점수를 계산하고, 구간을 매핑하며,
/// 부모 친화적 안심 메시지를 생성한다.
class ScoreCalculator {
  static const int totalQuestions = 18;
  static const int maxScorePerQuestion = 4;
  static const int maxTotalScore = totalQuestions * maxScorePerQuestion; // 72

  /// 6개 발달 영역 정의
  static const List<String> domains = [
    'physical',    // 신체·성장 → "몸 움직임"
    'sensory',     // 감각·지각 → "감각 반응"
    'cognitive',   // 인지·주의 → "집중과 관심"
    'language',    // 언어·의사소통 → "소리와 표현"
    'emotional',   // 정서·사회 → "마음과 관계"
    'regulation',  // 조절·수면·생리 → "생활 리듬"
  ];

  /// 영역 내부 ID → 부모 친화적 표시 이름 매핑
  static const Map<String, String> domainDisplayNames = {
    'physical': '몸 움직임',
    'sensory': '감각 반응',
    'cognitive': '집중과 관심',
    'language': '소리와 표현',
    'emotional': '마음과 관계',
    'regulation': '생활 리듬',
  };

  /// 응답 데이터로부터 총점을 계산한다.
  /// [responses]: key="domain_index" (예: "physical_0"), value=0~4
  static int calculateTotalScore(Map<String, int> responses) {
    return responses.values.fold(0, (sum, score) => sum + score);
  }

  /// 총점으로부터 퍼센트를 계산한다.
  static double calculatePercentage(int totalScore) {
    return (totalScore / maxTotalScore) * 100;
  }

  /// 퍼센트로부터 5구간 티어를 반환한다.
  /// 1 = 85~100% (매우 안정)
  /// 2 = 70~84% (안정적)
  /// 3 = 55~69% (보통)
  /// 4 = 40~54% (관찰 필요)
  /// 5 = 0~39% (주의 관찰)
  static int calculateTier(double percentage) {
    if (percentage >= 85) return 1;
    if (percentage >= 70) return 2;
    if (percentage >= 55) return 3;
    if (percentage >= 40) return 4;
    return 5;
  }

  /// 영역별 점수를 계산한다.
  /// 각 영역 3문항 × 4점 = 12점 만점
  static Map<String, int> calculateDomainScores(Map<String, int> responses) {
    final Map<String, int> domainScores = {};

    for (final domain in domains) {
      int score = 0;
      for (int i = 0; i < 3; i++) {
        final key = '${domain}_$i';
        score += responses[key] ?? 0;
      }
      domainScores[domain] = score;
    }

    return domainScores;
  }

  /// 티어에 따른 부모 안심 메시지를 반환한다.
  /// 원칙: 숫자보다 말로 안심시킨다. 죄책감을 주지 않는다.
  static String getTierMessage(int tier) {
    switch (tier) {
      case 1:
        return '오늘 관찰한 결과, 전반적으로 매우 안정적인 흐름이에요.\n'
               '지금 하고 계신 방식이 아기에게 잘 맞고 있어요.';
      case 2:
```

### 6-3. 관찰 기록 해석 로직 (ObservationInterpreter)

```dart
/// 활동 후 관찰 기록을 분석하여 4단계 피드백을 생성한다.
class ObservationInterpreter {
  /// 해석 레벨 정의
  /// 0 = 무리 없음
  /// 1 = 짧게 조정 권장
  /// 2 = 며칠 이어서 관찰
  /// 3 = 전문가 상담 고려

  /// 1단계 + 2단계 응답을 기반으로 해석 레벨을 계산한다.
  static int interpret({
    required Map<String, int> step1Responses,
    required Map<String, int> step2Responses,
  }) {
    // 1단계 점수 합산 (각 항목 0~2, 7항목 = 최대 14점)
    final step1Total = step1Responses.values.fold(0, (sum, v) => sum + v);
    final step1Max = step1Responses.length * 2;
    final step1Ratio = step1Max > 0 ? step1Total / step1Max : 0.5;

    // 2단계 점수 합산 (각 항목 0~2, 7항목 = 최대 14점)
    final step2Total = step2Responses.values.fold(0, (sum, v) => sum + v);
    final step2Max = step2Responses.length * 2;
    final step2Ratio = step2Max > 0 ? step2Total / step2Max : 0.5;

    // 종합 비율
    final overallRatio = (step1Ratio + step2Ratio) / 2;

    if (overallRatio >= 0.75) return 0;  // 무리 없음
    if (overallRatio >= 0.50) return 1;  // 짧게 조정
    if (overallRatio >= 0.30) return 2;  // 며칠 이어서
    return 3;                             // 전문가 상담 고려
  }

  /// 해석 레벨에 따른 메시지를 반환한다.
  static String getMessage(int level) {
    switch (level) {
      case 0:
        return '무리 없이 잘 진행된 것 같아요.\n'

    }
  }
```

### 6-4. 위험 신호 패턴 감지 (DangerSignAnalyzer)

```dart
/// 위험 신호 기록의 패턴을 분석하여 알림 메시지를 생성한다.
class DangerSignAnalyzer {
  /// 최근 N일간 위험 신호 기록을 분석한다.
  /// 3일 연속 같은 항목이 체크되면 추적 알림 트리거
  static DangerSignAnalysis analyze(List<DangerSignRecord> records) {
    if (records.isEmpty) {
      return DangerSignAnalysis(
        hasConsecutiveSigns: false,
        message: '',
        consecutiveDays: 0,
      );
    }

    // 최근 7일 기록 필터
    final now = DateTime.now();
    final recentRecords = records.where(
      (r) => now.difference(r.date).inDays <= 7
    ).toList()
      ..sort((a, b) => b.date.compareTo(a.date));

    // 연속 체크 분석
    int maxConsecutive = 0;
    String? consecutiveSign;

    if (recentRecords.length >= 3) {
      // 각 위험 신호 항목별 연속 일수 계산
      final allSignKeys = recentRecords.first.signs.keys;
      for (final signKey in allSignKeys) {
        int consecutive = 0;
        for (final record in recentRecords) {
          if (record.signs[signKey] == true) {
            consecutive++;
          } else {
            break;
          }
        }
        if (consecutive > maxConsecutive) {
          maxConsecutive = consecutive;
          consecutiveSign = signKey;
        }
      }
    }

    if (maxConsecutive >= 3) {
      return DangerSignAnalysis(
        hasConsecutiveSigns: true,
        message: '${maxConsecutive}일 연속 비슷한 패턴이 관찰되고 있어요.\n'
                 '걱정이 되시면, 소아과 방문 시 이 기록을 보여주세요.',
        consecutiveDays: maxConsecutive,
        signKey: consecutiveSign,
      );
    }

    return DangerSignAnalysis(
      hasConsecutiveSigns: false,
      message: '',
      consecutiveDays: maxConsecutive,
    );
  }
}

class DangerSignAnalysis {
  final bool hasConsecutiveSigns;
  final String message;
  final int consecutiveDays;
  final String? signKey;

  DangerSignAnalysis({
    required this.hasConsecutiveSigns,
    required this.message,
    required this.consecutiveDays,
    this.signKey,
  });
}
```

```dart
      now.day,
      time.hour,
      time.minute,

      scheduled = scheduled.add(const Duration(days: 1));
```

### 9-1. 아키텍처 개요

```dart
/// 현재 활성 아기 프로필 (sqflite 조회 + ref.invalidate()로 수동 새로고침)
@riverpod
class ActiveBaby extends _$ActiveBaby {
  @override
  Future<Baby?> build() async {
    final dao = ref.watch(babyDaoProvider);
    final settings = ref.watch(settingsProvider);

    if (settings.activeBabyId != null) {
      return await dao.getById(settings.activeBabyId!);
    }

    // 활성 아기가 지정되지 않았으면 첫 번째 아기 반환

/// 오늘의 데일리 기록 (sqflite 조회 — DB 변경 후 ref.invalidate()로 수동 새로고침)
@riverpod
class TodayRecord extends _$TodayRecord {
  @override
  Future<DailyRecord?> build() async {
    final baby = ref.watch(activeBabyProvider).valueOrNull;
    if (baby == null) return null;

    final dao = ref.watch(dailyRecordDaoProvider);
    return await dao.getTodayRecord(baby.id!);
  }

  Future<void> incrementFeeding() async {
    final record = await _getOrCreateTodayRecord();
    final dao = ref.read(dailyRecordDaoProvider);
    await dao.update(record.copyWith(
      feedingCount: record.feedingCount + 1,
      updatedAt: DateTime.now(),
    ));
    ref.invalidateSelf();
  }

  Future<void> incrementDiaper() async {
    final record = await _getOrCreateTodayRecord();
    final dao = ref.read(dailyRecordDaoProvider);
    await dao.update(record.copyWith(
      diaperCount: record.diaperCount + 1,
      updatedAt: DateTime.now(),
    ));
    ref.invalidateSelf();
  }

  Future<void> updateSleep(double hours) async {
    final record = await _getOrCreateTodayRecord();
    final dao = ref.read(dailyRecordDaoProvider);
    await dao.update(record.copyWith(
      sleepHours: hours,
      updatedAt: DateTime.now(),
    ));
    ref.invalidateSelf();
  }

  Future<void> updateMemo(String memo) async {
    final record = await _getOrCreateTodayRecord();
    final dao = ref.read(dailyRecordDaoProvider);
    await dao.update(record.copyWith(
      memo: memo,
      updatedAt: DateTime.now(),
    ));
    ref.invalidateSelf();
  }

  Future<DailyRecord> _getOrCreateTodayRecord() async {
    final baby = ref.read(activeBabyProvider).valueOrNull!;
    final dao = ref.read(dailyRecordDaoProvider);

    final existing = await dao.getTodayRecord(baby.id!);
    if (existing != null) return existing;

    final now = DateTime.now();
    final newRecord = DailyRecord(
      babyId: baby.id!,
      date: now,
      createdAt: now,
    );
    final id = await dao.insert(newRecord);
    return newRecord.copyWith(id: id);
  }

@riverpod

  final activities = ref.watch(currentActivitiesProvider);
  final baby = ref.watch(activeBabyProvider).valueOrNull;
  if (activities.isEmpty || baby == null) return null;

  // sqflite DAO로 오늘 완료한 활동 ID 조회
  final dao = ref.watch(activityRecordDaoProvider);
  final todayCompleted = await dao.getTodayCompletedActivityIds(baby.id!);
  final completedSet = todayCompleted.toSet();

  // 완료하지 않은 첫 번째 활동 추천
  final uncompleted = activities.where(
    (a) => !completedSet.contains(a.id),
  );
  if (uncompleted.isNotEmpty) return uncompleted.first;

  // 모두 완료했으면 첫 번째 활동 표시
  return activities.first;
}

/// 활동 타이머 상태
@riverpod
class ActivityTimer extends _$ActivityTimer {
  Timer? _timer;

  @override
  TimerState build() {
    ref.onDispose(() => _timer?.cancel());
    return TimerState.initial();
  }

  void start(int totalSeconds) {
    state = TimerState(
      totalSeconds: totalSeconds,
      remainingSeconds: totalSeconds,
      isRunning: true,
      isCompleted: false,
    );

    _timer = Timer.periodic(const Duration(seconds: 1), (timer) {
      if (state.remainingSeconds <= 1) {
        timer.cancel();
        state = state.copyWith(
          remainingSeconds: 0,
          isRunning: false,
          isCompleted: true,
        );
        // 진동 알림 (무음 모드)
        HapticFeedback.heavyImpact();
      } else {
        state = state.copyWith(
          remainingSeconds: state.remainingSeconds - 1,
        );
      }
    });
  }

  void pause() {
    _timer?.cancel();
    state = state.copyWith(isRunning: false);
  }

  void resume() {
    start(state.remainingSeconds);
  }

  void reset(int totalSeconds) {
    _timer?.cancel();
    state = TimerState.initial(totalSeconds: totalSeconds);
  }
}
```

> **sqflite 수동 새로고침 패턴 안내:**
> drift는 `.watch()` / `.watchSingleOrNull()`으로 DB 변경 시 자동으로 UI를 업데이트했지만,
> sqflite는 자동 스트림을 지원하지 않습니다. 대신 다음 패턴을 사용합니다:
> - DB에 데이터를 쓴 후 `ref.invalidateSelf()` 또는 `ref.invalidate(관련Provider)`를 호출
> - Riverpod이 Provider를 다시 빌드하여 최신 데이터를 조회
> - UI는 `ref.watch()`로 자동 반영됨
>
> 이 패턴은 drift의 스트림 방식보다 약간 더 명시적이지만, 코드 생성 없이 동작하며 analyzer 버전 충돌 문제가 없습니다.
