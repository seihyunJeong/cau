**버전:** v1.0
**작성일:** 2026-03-25
**프로젝트명:** 하루 한 가지, 0~60개월 발달 카드
**저자:** 전선혜, 김윤주

## 1. 프로젝트 개요

### 1-1. 비전

> "나보다 내 아이를 더 잘 아는 주치의를 주머니에 넣고 다니는 기분"

단순 육아 정보 앱이 아닌, **전문가의 발달 프로그램을 데이터 기반으로 제공하는 맞춤형 솔루션**.

### 1-2. 앱 목적

- 0~60개월 영유아의 발달을 **주 단위**로 추적·관찰
- 부모에게 **짧고 실천 가능한 신체활동 가이드** 제공 (하루 1~3분)
- 발달 반사 체크 및 6개 영역 관찰표를 통한 **인터랙티브 발달 스크리닝**
- 관찰 데이터를 자동 분석해 **맞춤형 솔루션** 리포트 제공
- 부모의 기록 데이터를 축적하여 **영유아 발달 연구 빅데이터** 확보

### 1-3. 타겟 유저

| 구분 | 설명 |
|---|---|
| **1차 유저** | 0~60개월 영유아를 키우는 부모(보호자) |
| **핵심 페르소나** | 신생아~영아기(0-4주) 초보 부모, 과학적 육아를 원하는 부모 |
| **2차 유저** | 조부모, 산후조리원, 영유아 관련 전문가 |

## 2. 콘텐츠 구조

### 2-1. 전체 구조

PDF 원고는 **주차별 챕터** 구조로, 각 챕터가 동일한 5단계 프레임워크를 따른다.

```
제N장: [주차] 신체활동 루틴 — "[주차 테마]"
  ├── N-1. 이 시기 한눈에 보기 (시기 개요)
  ├── N-2. 발달 포인트 (발달 특징 + 주의사항)
  ├── N-3. 활동 패키지 (5개 활동)
  │     ├── 활동 1~5 각각:
  │     │     ├── 활동 유형 / 연결 반사
  │     │     ├── 이렇게 해보세요 (단계별 가이드)
  │     │     ├── 관찰 포인트
  │     │     ├── 왜 이 활동을 하나요? (근거)
  │     │     ├── 기대 효과 / 주의사항 / 활용 TIP
  │     │     └── 필요 교구 / 준비물 + 대체 DIY
  ├── 부록 1. 활동 후 자연 반응 관찰표
  ├── N-4. 부모 관찰 기록용 체크리스트 (6영역 × 3문항)
  └── 부록 2. 각 문항 출처 도구 (비공개)
```

### 2-2. 현재 콘텐츠 범위

| 챕터 | 주차 | 테마 | 활동 수 |
|---|---|---|---|
| 제1장 | 0-1주 | "신경이 안정되는 시간" | 5개 |
| 제2장 | 2-3주 | "연결이 늘어나는 시간" | 5개 |
| *(향후 확장)* | *4주~60개월* | *미정* | *주차별 5개* |

### 2-3. 6개 발달 관찰 영역

| # | 영역 | 관찰 내용 |
|---|---|---|
| 1 | 신체·성장 | 자세, 근긴장, 활력, 체중 변화 |
| 2 | 감각·지각 | 빛·소리·촉감 반응의 적절성 |
| 3 | 인지·주의 | 얼굴/빛 주의 머무름, 자극 조절 |
| 4 | 언어·의사소통 | 울음 분화, 목소리 반응 |
| 5 | 정서·사회 | 접촉 진정, 표정 부드러움 |
| 6 | 조절·수면·생리 | 잠-깸 전환, 수유 후 진정 |

### 2-4. 점수 체계

- 18문항 × 4점 만점 = **총 72점**
- 퍼센트 = (획득 점수 / 72) × 100
- 5구간 해석: 85~100% / 70~84% / 55~69% / 40~54% / 0~39%
- 각 구간별 **부모 솔루션 메시지** 자동 제공

## 3. 서비스 기능 정의

### 3-1. 메인 화면 (대시보드) — "오늘 우리 아이는?"

| 요소 | 설명 |
|---|---|
| **아기 프로필** | 이름, 생년월일, 현재 주차 자동 계산 표시 |
| **오늘의 미션 카드** | 해당 주차의 활동 중 오늘 추천 활동 1개를 카드로 표시 |
| **성장 요약** | 최근 입력된 체중/키 기반 미니 그래프 + 안심 메시지 |
| **관찰 점수 요약** | 최근 체크리스트 결과를 6개 영역 레이더 차트로 표시 |
| **빠른 기록** | 수유/배변/수면 원탭 기록 버튼 |

### 3-2. 데일리 기록 — "우리 아이 하루 기록"

| 기록 항목 | 입력 방식 | 시각화 |
|---|---|---|
| 체중 (kg) | 숫자 입력 | 성장 곡선 그래프 (WHO 기준선 오버레이) |
| 키 (cm) | 숫자 입력 | 성장 곡선 그래프 |
| 머리 둘레 (cm) | 숫자 입력 | 추이 그래프 |
| 수면 시간 | 시간 입력 or 타이머 | 일별/주별 막대 그래프 |
| 수유 횟수 | 카운터 (+1 탭) | 일별 추이 |
| 배변 횟수 | 카운터 (+1 탭) | 일별 추이 |
| 부모 메모 | 자유 텍스트 | 타임라인 형태 |

- 입력 시 즉시 **안심/주의 메시지** 표시
- 누적 그래프에 **WHO Growth Standards** 기준선 오버레이
- 주간/월간 리포트 자동 생성

### 3-3. 주차별 활동 가이드 — "하루 한 가지씩 실천해 보세요!"

**활동 카드:** 각 활동을 카드 형태로 제공 (사진 + 유형 + 권장 시간 + 시작/상세 버튼)

**활동 상세 화면:**

| 섹션 | 내용 |
|---|---|
| 단계별 가이드 | "이렇게 해보세요" 1~5단계, 스텝 바이 스텝 카드 + 일러스트 |
| 타이머 | 권장 시간 카운트다운, 활동 중 안내 문구 순차 표시 |
| 관찰 포인트 | 활동 중 체크할 관찰 항목 |
| 근거 설명 | "왜 이 활동을 하나요?" (접힌 상태 → 펼치기) |
| 기대 효과 | 기대할 수 있는 변화 |
| 주의사항 | 하지 말아야 할 것 (경고 카드) |
| 활용 TIP | 실천 팁 |
| 필요 교구 | 준비물 + 대체/DIY + 구매 링크 |

**타이머 동작:**
- 활동별 권장 시간 프리셋 (20초, 30초, 40초 등)
- 타이머 종료 시 부드러운 알림음
- 활동 중 안내 문구 순차 표시 (예: "5초 조용히 멈춰주세요")
- 완료 시 관찰 기록 화면으로 자동 전환

### 3-4. 활동 후 자연 반응 관찰표

**1단계:** 전체 상태 관찰 — "활동 후 편안해 보였어요" 등 7~8항목 (예/보통/아니오)
**2단계:** 자연 반응 관찰 — 모로반사, 탐색반사 등 반응 체크 (뚜렷함/약하게/잘 안 보임)
**기록 메모:** 편안했던 활동, 부담스러웠던 활동, 조정 방향 선택
**자동 해석:** 4단계 피드백 (무리 없음 / 짧게 조정 / 며칠 이어서 / 상담 고려)

### 3-5. 발달 스크리닝 (부모 관찰 체크리스트)

- 6개 영역 × 3문항 = **18문항** (0~4점 5단계)
- 각 문항에 메모란 제공
- 점수 기준 안내 상단 고정 (4=자주 안정적 ~ 0=거의 안 보임)

**결과 화면:**
- 총점/퍼센트 자동 계산 + 5구간 매핑
- 6개 영역 레이더 차트 시각화
- 구간별 부모 솔루션 메시지 자동 표시
- 주간/월간 추이 그래프

### 3-6. 위험 신호 알림

- 6개 위험 신호 항목 중 1개 이상 체크 시 주의 배너 표시
- "2~3일 이상 지속되면 전문가 상담을 고려해보세요" 안내
- 전문가 연결 또는 병원 찾기 기능 제공

### 3-7. 교구 매칭 큐레이션

| 기능 | 설명 |
|---|---|
| 교구 카드 | 활동별 필요 교구를 사진 + 설명으로 표시 |
| 대체/DIY 가이드 | "별도 교구 없이 가능", "집에 있는 OO로 대체" 안내 |
| 구매 링크 | 추천 제품으로 바로 연결 (커머스 연동) |

### 3-8. 푸시 알림

| 알림 유형 | 예시 | 트리거 |
|---|---|---|
| 주차 전환 | "오늘부터 2-3주차예요! 새로운 활동을 확인해보세요" | 생년월일 기반 자동 |
| 데일리 미션 | "오늘의 활동: 미니 시선 산책 (10~20초)" | 매일 설정 시간 |
| 기록 리마인더 | "오늘 하루 기록을 아직 안 하셨어요" | 미입력 시 저녁 |
| 성장 마일스톤 | "우리 아기가 태어난 지 14일째!" | 생년월일 기반 |
| 위험 신호 추적 | "3일 연속 조절·수면 점수가 낮아요" | 데이터 패턴 감지 |

## 5. 핵심 사용자 플로우

### 5-1. 첫 진입 (온보딩)

```
앱 설치 → 회원가입/로그인

```

```
  → 앱 진입 → 활동 카드 탭
  → 단계별 가이드 확인

  → 타이머 종료 → 관찰 기록 화면
  → 1단계 전체 상태 체크 (7항목)
  → 2단계 자연 반응 체크 (7항목)
  → 오늘 메모 작성
  → 해석 결과 확인
  → 홈으로 복귀
```

### 5-3. 주간 체크 (발달 스크리닝)

```
주 1회 알림 ("이번 주 발달 체크를 해보세요")
  → 발달 체크 탭 진입
  → 6개 영역 × 3문항 = 18문항 순차 입력 (0~4점)
  → 위험 신호 6항목 체크
  → 자동 분석 결과 화면
  → 총점/퍼센트/구간/솔루션 확인
  → 레이더 차트 + 주간 추이 확인
  → (위험 신호 있을 시) 전문가 상담 안내
```

## 6. UX 원칙 — "엄마가 쓰는 앱"

> 이 앱의 핵심 유저는 신생아를 안은 채 한 손으로 앱을 여는 엄마다.
> 잠이 부족하고, 불안하고, "내가 잘하고 있는 건지" 확인받고 싶은 상태다.
> 앱은 할 일을 늘려주는 도구가 아니라, **"오늘 이것만 하면 돼"라고 말해주는 도구**여야 한다.

### 6-1. 핵심 UX 원칙 5가지

#### 원칙 1: 하나만 해도 충분하다

PDF 원고의 반복 메시지: *"하루에 전부 하지 않아도 괜찮아요. 1~2개만, 1-3분이면 충분합니다."*

| 적용 | 나쁜 예 | 좋은 예 |
|---|---|---|
| 홈 화면 | 활동 5개 + 기록 7개 + 체크리스트 한꺼번에 노출 | **"오늘의 활동 1개"** 카드만 크게 표시, 나머지는 스크롤 아래 |
| 활동 완료 후 | "관찰 기록도 하시겠어요? 체크리스트도 하시겠어요?" | 활동 완료 축하 → **"기록하기"는 선택**, 안 해도 완료 처리 |
| 데일리 기록 | 7개 항목 빈칸 모두 노출 | **수유 횟수 하나만 탭**해도 오늘 기록 완료로 인정 |

#### 원칙 2: 숫자보다 말로 안심시킨다

부모는 "72점 만점에 58점"보다 **"오늘 이 정도면 괜찮아요"**가 필요하다.

| 적용 | 나쁜 예 | 좋은 예 |
|---|---|---|
| 체크리스트 결과 | "80.6% — 대체로 안정적인 편" (숫자가 먼저) | **"오늘 관찰한 결과, 대체로 안정적인 흐름이에요."** (말이 먼저, 숫자는 작게) |
| 영역별 점수 | "신체·성장: 10/12 (83%)" | **"몸 움직임이 편안해 보여요"** + 막대 그래프 (숫자 없이 시각적) |
| 낮은 점수 | "40~54% — 루틴 조정 및 재확인 권장" | **"오늘은 조금 예민한 날이었을 수 있어요. 내일 한 번 더 살펴볼까요?"** |
| 위험 신호 | "위험 신호 감지됨" (빨간 경고) | **"며칠 더 지켜본 뒤, 걱정이 계속되면 전문가와 이야기해 보세요."** |

#### 원칙 3: 전문 용어를 부모 언어로 번역한다

PDF에는 전문 근거(NBAS, 모로반사 등)가 필요하지만, 앱 화면에서는 부모가 바로 이해할 수 있는 말로 바꾼다.

| 전문 용어 (PDF/내부) | 앱 화면 표현 |
|---|---|
| 모로반사(Moro) | 깜짝 놀라며 팔을 벌리는 반응 |
| 탐색반사(Rooting) | 입 주변을 건드리면 고개를 돌리는 반응 |
| 파악반사(Palmar) | 손바닥을 건드리면 꽉 쥐는 반응 |
| 각성 시간 | 깨어 있는 시간 |
| 공동조절(Co-regulation) | 부모가 안아주며 함께 안정되는 것 |
| 과자극 | 자극이 너무 많아 지친 상태 |
| 굴곡 자세 | 팔다리를 몸 쪽으로 모은 자세 |
| 신경계 통합 | 아기의 뇌와 몸이 안정을 배우는 과정 |
| consolability | 달래주면 진정되는 정도 |

> 전문 용어는 "더 알아보기" 접힘 영역에 배치. 메인 화면에는 절대 노출하지 않는다.

#### 원칙 4: 한 손, 30초 안에 끝낼 수 있어야 한다

신생아 엄마의 현실: 아기를 안은 채 한 손으로 앱을 열고, 아기가 울기 전에 끝내야 한다.

| 적용 | 규칙 |
|---|---|
| **모든 입력** | 키보드 타이핑 최소화. 탭, 슬라이더, 카운터(+1) 위주 |
| **빠른 기록** | 홈 화면에서 **원탭**으로 수유/배변/수면 기록 (상세 입력은 나중에) |
| **활동 완료** | 타이머 종료 → "완료" 버튼 1번이면 기록됨. 관찰 기록은 선택 |
| **체크리스트** | 18문항이지만 **한 문항당 탭 1번** (0~4 중 선택). 중간 저장 지원 |
| **글씨 크기** | 최소 16px. 어두운 방에서 낮은 밝기로도 읽을 수 있게 |
| **터치 영역** | 버튼/선택지 최소 48×48dp. 한 손 엄지로 닿는 범위 내 배치 |

#### 원칙 5: 죄책감을 주지 않는다

이 앱의 가장 중요한 원칙. 기록을 안 했다고, 점수가 낮다고, 활동을 건너뛰었다고 **절대 부정적 피드백을 주지 않는다.**

| 상황 | 나쁜 예 | 좋은 예 |
|---|---|---|
| 3일째 기록 없음 | "3일 동안 기록이 없습니다!" | (조용히 다음 접속 시) **"다시 오셨네요. 오늘부터 편하게 시작해요."** |
| 활동 건너뜀 | "이번 주 활동 완료율 20%" | **표시하지 않음.** 완료한 활동만 체크 표시 |
| 체크리스트 낮은 점수 | "주의가 필요합니다" | **"오늘 아기가 조금 예민했을 수 있어요. 며칠 더 지켜보면서 리듬을 찾아가요."** |
| 위험 신호 체크 | "경고: 즉시 병원 방문 권장" | **"걱정이 되신다면, 다음 소아과 방문 시 이 기록을 보여주세요."** |
| 연속 기록 끊김 | "스트릭이 끊어졌습니다" | 스트릭 **끊김 표시 없음**. 다시 시작하면 "다시 시작했어요!" |

### 6-2. 입력 최소화 설계

기획서의 기능은 "할 수 있는 최대"를 정의한 것이고, 실제 UX에서는 **"하지 않아도 되는 것"을 명확히** 해야 한다.

| 기능 | 필수 입력 | 선택 입력 | 안 해도 되는 것 |
|---|---|---|---|
| **데일리 기록** | 아무 항목 1개 (수유 or 배변 등) | 체중, 키, 메모 | 7개 전부 채우기 |
| **활동** | [완료] 버튼 1번 | 타이머 사용, 관찰 기록 | 하루 5개 전부 하기 |
| **활동 후 관찰** | 1단계 전체 상태 3~4항목 | 2단계 반사 관찰, 메모 | 매번 기록하기 |
| **발달 체크리스트** | 18문항 각 1탭 | 문항별 메모 | 매주 하기 (2주에 1번도 OK) |
| **위험 신호** | 해당 시에만 체크 | 메모 | 매번 확인하기 |

### 6-3. 용어 통일 가이드 (앱 내 표현)

| 카테고리 | 내부/기획 용어 | 앱에서 보이는 표현 |
|---|---|---|
| 탭 이름 | 대시보드 | **홈** |
| 탭 이름 | 데일리 기록 | **기록** |
| 탭 이름 | 활동 가이드 | **활동** |
| 탭 이름 | 발달 스크리닝 | **발달 체크** |
| 기능 | 활동 후 자연 반응 관찰표 | **오늘 아기는 어땠나요?** |
| 기능 | 부모 관찰 기록용 체크리스트 | **이번 주 발달 살펴보기** |
| 결과 | 점수 해석 가이드 | **오늘의 관찰 결과** |
| 결과 | 위험 신호 | **함께 살펴볼 점** |
| 결과 | 부모 솔루션 | **이렇게 해보세요** |
| 버튼 | 활동 시작 | **시작하기** |
| 버튼 | 관찰 기록 제출 | **기록 완료** |
| 버튼 | 체크리스트 제출 | **결과 보기** |
| 상태 | 미완료 | 표시하지 않음 (완료한 것만 체크) |
| 상태 | 미기록 | **"언제든 기록할 수 있어요"** |

### 6-4. 접근성 고려

| 항목 | 기준 |
|---|---|
| **어두운 방** | 야간 수유 시 눈부심 방지: 다크 모드 또는 자동 밝기 감소 |
| **한 손 조작** | 주요 버튼은 화면 하단 1/3 영역에 배치 (엄지 도달 범위) |
| **소리 없는 환경** | 아기가 자는 동안 사용: 기본 무음, 진동 알림. 타이머 종료 시 화면 깜빡임으로 알림 |
| **빠른 이탈** | 아기가 울면 즉시 앱을 떠남: 중간 저장 자동, 다음 접속 시 이어서 |
| **조부모 모드** | 글씨 크기 확대, 핵심 기능만 노출, "오늘의 활동" 카드 1장만 표시 |

## 7. 디자인 가이드라인

### 7-1. 톤 앤 무드

- **따뜻하고 안정감 있는** 색감 (PDF 원본의 크림/베이지 톤 계승)
- **부드러운 일러스트** 스타일 (아이콘, 활동 가이드)
- **과하지 않은 정보량** — 한 화면에 핵심만 ("짧게, 부드럽게"라는 앱 철학 반영)
- **불안을 유발하지 않는** 표현 — 점수가 낮아도 "문제"가 아니라 "관찰 포인트"로 안내

## 11. 법적 고려사항

### 11-1. 의료기기법 / 건강정보 관련

| 항목 | 대응 |
|---|---|
| **의료기기 해당 여부** | 본 앱은 진단·치료 목적이 아닌 **부모 관찰 기록 보조 도구**이므로 의료기기에 해당하지 않음. 단, "진단", "검사", "치료" 등의 표현을 사용하지 않도록 주의. |
| **면책 문구 필수** | 앱 내 모든 체크리스트/리포트 화면에 다음 문구 명시: *"본 관찰표는 표준화 검사가 아닌 부모 관찰 기록용 구조입니다. 의학적 진단을 대체하지 않습니다."* |
| **전문가 자문 표시** | 콘텐츠 출처 명시: "본 콘텐츠는 NBAS, Bayley-4, ASQ-3 등을 참고하여 전문가가 설계하였습니다" |
| **위험 신호 안내** | "전문가 상담을 고려해보세요"로 안내하되, 직접 의료 조언을 제공하지 않음 |

### 11-2. 개인정보보호법 (아동 데이터)

| 항목 | 대응 |
|---|---|
| **개인정보 수집 동의** | 회원가입 시 필수 동의: 아기 이름, 생년월일, 성장 기록 |
| **만 14세 미만 아동 정보** | 아기 본인이 아닌 **법정대리인(부모)**이 입력하므로, 부모의 동의로 처리. 개인정보보호법 제22조의2 준수 |
| **민감정보 처리** | 건강 관련 데이터(체중, 발달 점수)는 **민감정보**에 해당할 수 있음 → 별도 동의 항목 분리 |
| **데이터 보관/삭제** | 회원 탈퇴 시 즉시 삭제, 보관 기간 명시 (예: 탈퇴 후 30일 이내 파기) |
| **데이터 암호화** | 전송 시 TLS 1.3, 저장 시 AES-256 암호화 |
| **제3자 제공** | 기본적으로 비제공. 연구 목적 활용 시 별도 옵트인 동의 |

### 11-3. 연구 데이터 활용

| 항목 | 대응 |
|---|---|
| **비식별화** | 연구 데이터는 반드시 **비식별 처리** 후 활용 (이름, 생년월일 등 직접 식별자 제거) |
| **IRB 심의** | 데이터를 학술 연구에 활용할 경우 기관생명윤리위원회(IRB) 심의 필수 |
| **옵트인 동의** | 앱 내 별도 화면: "연구 참여에 동의하시겠습니까?" (거부 시 앱 사용에 영향 없음) |
| **동의 철회** | 언제든 연구 참여 동의를 철회할 수 있으며, 기존 데이터도 삭제 요청 가능 |

### 11-4. 기타 법적 사항

| 항목 | 대응 |
|---|---|
| **이용약관** | 서비스 이용 조건, 책임 한계, 환불 정책 명시 |
| **전자상거래법** | 교구 커머스 연동 시 통신판매업 신고, 청약철회 규정 준수 |
| **저작권** | PDF 원고 콘텐츠의 저작권 귀속 명시, 앱 내 무단 복제 방지 |
| **앱스토어 심사** | Apple 건강 관련 앱 가이드라인 준수 (헬스킷 연동 시 추가 심사) |

| 앱스토어 심사 리젝 | 중 | 중 | 건강 관련 앱 가이드라인 사전 검토, 면책 문구 및 전문가 검증 근거 준비 |
