# Design Review v3: "하루 한 가지" 앱 디자인 수정 최종 재평가

**평가일:** 2026-03-27
**평가 모드:** 다크 모드 (Dark Mode) -- 다크 모드 토글 비작동(ON 고정)으로 다크 모드에서 평가
**평가 기준:** Design Quality / Originality / Craft / Functionality (각 1-10점)
**평가 대상:** 전체 7개 화면 (홈, 기록, 활동 목록, 활동 상세, 발달 체크, 마이, 타이머)
**평가 방법:** Marionette MCP 실시간 접속 + 코드 리뷰
**이전 리뷰:** design-review-v2.md (라이트 모드 평가)

**중요:** 이번 평가는 다크 모드에서 진행되었으므로, v2(라이트 모드)와의 직접 비교에는 모드 차이를 감안해야 한다. 다만, v2에서 "코드로만 확인"했던 다크 모드 대응이 실제로 렌더링되는 것을 이번 v3에서 처음으로 시각적으로 검증했다.

## 진단서 수정 항목 확인 (5개 기준)

### 1. 영역별 색상 구분 적용 여부

**판정: PASS**

**코드 확인:**
- `app_colors.dart`: 6개 영역별 전경색(domainHolding, domainSensory, domainSound, domainVision, domainTouch, domainBalance) + 6개 배경색(xxxBg) 추가 완료.
- `activity_type_chip.dart`: `domainColor()` / `domainBgColor()` static 메서드로 유형 문자열 기반 자동 매핑 구현.
- `activity_card.dart`: 좌측 4px 악센트바에 `ActivityTypeChip.domainColor(activity.type)` 적용.
- `today_mission_card.dart`: 상단 3px 그라데이션 스트라이프에 영역별 색상 적용.
- `activity_filter_chips.dart`: 선택 상태에서 영역별 색상(배경, 체크마크, 레이블, 테두리) 반영.

**스크린샷 확인 (Marionette):**
- 활동 목록 "전체" 보기: 안기 카드(로즈핑크 악센트바+칩), 감각 카드(그린 악센트바+칩), 소리 카드(스카이블루 악센트바+칩) -- 세 영역이 시각적으로 명확히 구분됨.
- "안기" 필터 선택 시: 필터 칩이 로즈핑크 색상, 카드 악센트바도 로즈핑크.
- "감각" 필터 선택 시: 필터 칩이 그린 색상, 카드 악센트바도 그린.
- 홈 화면 오늘의 미션 카드: "소리" 유형 → 스카이블루 상단 스트라이프 표시.

**개선 정도:** v2에서 "모노크롬 오렌지"로 지적된 문제가 완전히 해결됨. 영역별 시각 구분이 즉각적이다.

### 2. 카드-배경 대비 개선 여부

**판정: PASS**

**코드 확인:**
- `app_theme.dart` lightTheme: CardTheme에 `BorderSide(color: AppColors.lightBeige, width: 1)` 추가.
- `app_theme.dart` darkTheme: CardTheme에 `BorderSide(color: AppColors.darkBorder, width: 1)` 유지.
- 개별 카드 위젯(counter_card, today_mission_card 등)에도 `isDark` 분기로 border 추가.

**스크린샷 확인 (Marionette):**
- 다크 모드에서 카드(darkCard: #2A231D)와 배경(darkBg: #1A1512)의 명도 차이가 충분하며, darkBorder(#3D3027) 테두리로 카드 경계가 명확하게 보인다.
- 기록 탭: 수유/배변 카드, 수면 카드, 성장 기록, 메모 카드 모두 배경과 구분됨.
- 마이 탭: 알림 설정, 화면 설정, 데이터 섹션 카드 모두 테두리로 그룹핑이 명확.

**개선 정도:** v2에서 "라이트 모드에서는 충분, 다크 모드 미확인"이었던 것이 이번에 다크 모드에서도 확인됨. 카드 경계가 뚜렷하다.

### 3. 타이포그래피 위계 개선 여부

**판정: PASS**

**코드 확인:**
- `app_text_styles.dart`: h1 = w700, h2 = w600, h3 = w500. 세 단계 weight 위계 형성 완료.
- 이전: h1=w600, h2=w600, h3=w600 (모두 동일).
- 현재: h1(w700, 22sp) > h2(w600, 18sp) > h3(w500, 16sp) -- weight와 size 모두 차별화.

**스크린샷 확인 (Marionette):**
- 홈 화면: "하루 한 가지" 앱 타이틀(h3, w500)과 "오늘의 활동" 섹션 제목(headlineMedium=h2, w600)의 무게감 차이가 식별 가능.
- 활동 목록: "하루 한 가지씩 실천해 보세요!" (h1, w700)이 카드 내 활동 제목(h3, w500)보다 시각적으로 무거움.
- 활동 상세: "품-숨-멈춤 루틴" 제목(h1, w700)이 본문 텍스트와 확실히 구분됨.

**개선 정도:** 섹션 제목과 카드 제목의 위계가 미세하지만 분명하게 느껴진다. 이전의 "모든 heading이 같은 weight" 문제 해결.

### 4. 섹션 간격 넓어졌는가

**판정: PASS**

**코드 확인:**
- `app_dimensions.dart`: sectionGap = 32 (이전: 24). cardGap = 12와 2.6배 차이.
- `home_screen.dart`: 모든 섹션 간 SizedBox에 `AppDimensions.sectionGap` 사용 (line 53, 62, 71).

**스크린샷 확인 (Marionette):**
- 홈 화면: "빠른 기록" -> "오늘의 활동" -> "성장 요약" 섹션 간 여백이 카드 간 여백보다 눈에 띄게 넓다.
- 마이 탭: "알림 설정" -> "화면 설정" -> "데이터" 섹션 간 간격이 균일하고 넓다.

**개선 정도:** 24->32는 33% 증가. 시각적으로 섹션 구분이 명확해졌다.

### 5. 그림자 강화 여부

**판정: PARTIAL**

**코드 확인:**
- `app_shadows.dart`: low 그림자 alpha 0x0D->0x14, blur 4->6, offset y 1->2. subtle 레벨 신규 추가.
- `today_mission_card.dart`: 라이트 모드에서 `AppShadows.medium` 적용 (다크 모드에서는 null).

**스크린샷 확인 (Marionette):**
- 다크 모드에서는 그림자가 배경에 흡수되어 시각적 효과가 미미함. 이는 다크 모드의 본질적 특성이므로 결함은 아니다.
- 카드 구분은 border에 의존하고 있으며, 이 방식이 다크 모드에서는 올바른 접근이다.

**개선 정도:** 코드상 그림자 강도가 증가했으나, 다크 모드에서는 border가 주된 구분 수단이므로 그림자 효과를 시각적으로 확인하기 어렵다. 라이트 모드 검증이 필요하지만 다크 모드 토글 비작동으로 불가.

## 잔여 이슈 (비차단)

### 1. 다크 모드 토글 비작동 (기능 버그, pre-existing)
- **심각도:** 중요하지만 디자인 범위 밖
- **현재 상태:** 마이 탭에서 "다크 모드" 토글을 OFF로 전환해도 라이트 모드로 바뀌지 않음.
- **영향:** 라이트 모드의 CardTheme border(lightBeige), 그림자 효과, cream/white 대비 등을 실제 화면에서 검증 불가.
- **라이트 모드 코드 리뷰:** CardTheme에 lightBeige border, AppShadows.medium 적용 등 코드 수준에서 올바르게 구현되어 있음.

### 2. 정보 배너 색상 (경미)
- **위치:** `lib/core/widgets/danger_sign_banner.dart:29`
- **현재:** `AppColors.softYellow.withValues(alpha: 0.2)` (주의/경고 색상)
- **진단서 권고:** 정보성 안내 문구에는 `AppColors.infoBg` (소프트 블루) 사용
- **상태:** `info`/`infoBg` 색상이 app_colors.dart에 추가되었으나 DangerSignBanner에는 미적용. 다만 해당 배너의 맥락("위험 신호 안내")이 softYellow와 의미적으로 일치하므로, 심각한 결함은 아님. 별도의 InfoBanner 위젯이 필요할 수 있음.

### 3. 관찰 폼 하드코딩 Color (경미)
- **위치:** `lib/features/activity/screens/observation_form_screen.dart:360`
- **현재:** `Color(0x1A000000)` 하드코딩
- **수정:** `AppShadows.medium`의 color 값과 유사하므로 AppShadows 상수로 교체 가능. 또는 최소한 AppColors에 shadow 색상 추가.
- **심각도:** 낮음. 그림자 색상이라 시각적 영향 미미.

### 4. 프로필 아바타 커스텀 일러스트 미적용 (범위 밖)
- **현재:** Icons.child_care 시스템 이모지 기반.
- **사유:** 에셋 제작 필요 (코드 범위 밖).
