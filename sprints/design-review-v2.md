# Design Review v2: "하루 한 가지" 앱 디자인 수정 재평가

**평가일:** 2026-03-27
**평가 모드:** 라이트 모드 (Light Mode) -- 다크 모드 토글 비작동으로 라이트 모드에서 평가
**평가 기준:** Design Quality / Originality / Craft / Functionality (각 1-10점)
**평가 대상:** 이전 FAIL 4개 화면 (홈, 기록, 발달 체크, 마이)

**참고:** 이전 리뷰(design-review.md)에서 다크 모드 평가였으나, 다크 모드 토글이 여전히 비작동하여 라이트 모드에서 평가. 코드 리뷰로 다크 모드 대응 코드의 존재는 확인함.

## 1. 홈 화면

### 이전 점수: 6.5 (FAIL)
### 이전 핵심 이슈: 카드 구분 약함, 빈 상태 빈약, 프로필 아바타 단순

### 스크린샷 평가

| 기준 | 이전 | 현재 | 코멘트 |
|---|---|---|---|
| Design Quality | 7 | 7 | 라이트 모드에서 따뜻한 cream 톤 배경(#FBF7F2)과 white 카드의 대비가 자연스럽다. 타이포그래피 위계가 명확하다. 카드 배경(white)과 화면 배경(cream)의 구분이 라이트 모드에서는 충분하다. 다크 모드에서는 CardTheme에 border가 추가되어 코드상 해결되었으나 실제 확인은 불가. |
| Originality | 6 | 7 | "시작하기" 버튼에 play_arrow 아이콘이 추가되어 CTA 성격이 더 명확해졌다. 빈 상태 카드에 warmOrange tinted 원형 아이콘 배경이 추가되어 시각적 focal point가 생겼다. 프로필 아바타는 여전히 이모지 기반이나 warmOrange 링이 추가되어 약간의 개성이 추가됨. |
| Craft | 7 | 카드 간 간격이 균일하다. "시작하기" 버튼에 elevation(2)과 warmOrange 40% shadow가 적용되어 입체감이 생겼다. 빈 상태 아이콘이 56dp로 커지고 원형 배경(80dp)으로 감싸져 존재감이 확보되었다. 서브 메시지("기록 탭에서 체중, 키를 입력해보세요")가 추가되어 빈 상태의 의도가 명확해졌다. |
| Functionality | 7 | 정보 밀도가 적절하다. 빠른 기록 버튼("수유 +1", "배변 +1")이 작동하고 카운터가 정상 증가한다. "시작하기" 버튼은 여전히 네비게이션 미구현(기능 이슈, 디자인 범위 밖). 빈 상태 서브 메시지가 사용자 행동을 안내한다. |
| **평균** | **6.5** | **7.0** | **PASS (경계선)** |

### 개선된 점
- 빈 상태 카드: 56dp 아이콘 + warmOrange tinted 원형 배경 + 서브 메시지 추가. 이전보다 훨씬 풍부하고 의도적인 빈 상태 표현.
- "시작하기" 버튼: play_arrow 아이콘 + elevation + shadow 추가로 CTA 성격 강화.
- 다크 모드 border: 코드에 CardTheme border와 개별 카드 border 일괄 추가 확인. (실제 다크 모드 시각 확인 불가)

### 잔여 이슈
- **"시작하기" 버튼 네비게이션 미구현:** 디자인 이슈가 아닌 기능 이슈. fix-result에서 "범위 밖"으로 명시.
- **프로필 아바타:** 여전히 시스템 이모지 기반. 커스텀 일러스트 미적용(에셋 제작 필요).
- **카드 배경/화면 배경 구분:** 라이트 모드에서는 white vs cream으로 충분하지만, 다크 모드 시각 검증 불가.

## 2. 기록 탭

### 이전 점수: 5.75 (FAIL)
### 이전 핵심 이슈: -/+ 버튼 스타일 불일치, 성장 기록 텍스트 가독성, 수면 입력 힌트 부족

### 스크린샷 평가

| 기준 | 이전 | 현재 | 코멘트 |
|---|---|---|---|
| Design Quality | 6 | 7 | 날짜 네비게이션 깔끔. 카운터 카드 구조가 일관적(emoji + label + 카운터). 수면 입력에 시계 아이콘과 dropdown arrow가 추가되어 인터랙션 힌트가 명확해졌다. "성장 기록 추가하기 +" 텍스트가 darkBrown 색상으로 변경되어 가독성 확보. |
| Originality | 5 | 6 | 카운터 UI는 여전히 표준 패턴이나, 통일된 warmOrange 테마의 버튼 스타일이 앱 고유 색감을 살린다. 수면 입력의 "예: 14시간 30분" 힌트가 맥락 정보를 제공한다. |
| Craft | 5 | 7 | **가장 크게 개선된 영역.** -/+ 버튼이 완전히 통일된 스타일로 변경: 활성 상태 = warmOrange 10% 배경 + warmOrange 40% border + warmOrange 아이콘, 비활성(count=0) = muted 30% 배경 + muted border. AnimatedContainer로 상태 전환이 부드럽다. "성장 기록 추가하기 +" 헤더 텍스트가 darkBrown으로 명확하게 보인다. 다크 모드용 배경/border 대응 코드 확인. |
| Functionality | 7 | 7 | 카운터 +/- 작동 정상. 수면 힌트 텍스트("탭해서 시간 선택 (예: 14시간 30분)")가 사용자에게 명확한 안내 제공. 성장 기록 확장 타일 정상 작동. |
| **평균** | **5.75** | **7.0** (소수점: 6.75 반올림) | **PASS (경계선)** |

### 개선된 점
- **-/+ 버튼 스타일 완전 통일:** 양쪽 버튼이 동일한 배경/border/아이콘 스타일을 공유. 비활성(count=0)일 때만 opacity 50% 처리. 이전의 "-버튼이 비활성으로 오인되는" 문제 해결.
- **수면 입력 힌트:** "탭해서 시간 선택 (예: 14시간 30분)" + 시계 아이콘 + dropdown 화살표 추가.
- **성장 기록 텍스트 가독성:** 다크 모드에서 darkTextPrimary로 오버라이드 (코드 확인).
- **다크 모드 border:** 모든 카드에 isDark 조건부 border 추가 (코드 확인).

### 잔여 이슈
- **카운터 버튼 비활성 상태 시각 차이:** 비활성(-) 버튼과 활성(+) 버튼 사이의 시각적 차이가 명확하나, 비활성 상태의 muted 색상이 약간 너무 연해서 버튼이 아닌 빈 공간으로 오인될 수 있다. border가 거의 안 보이는 수준. 이는 경미한 이슈.
- **카운터 카드 내부 여백:** 수유/배변 아이콘+텍스트가 중앙 정렬로 개선되었으나, 카드 세로 패딩이 약간 과하게 느껴진다(16dp 상하). 이는 취향의 문제.

## 3. 발달 체크 탭

### 이전 점수: 5.75 (FAIL)
### 이전 핵심 이슈: [치명적] 안심 메시지 텍스트 대비 문제, [심각] 레이더 차트 레이블 잘림

### 스크린샷 평가

| 기준 | 이전 | 현재 | 코멘트 |
|---|---|---|---|
| Design Quality | 6 | 8 | 안심 메시지 카드가 mint/cream 배경 위에 darkBrown 텍스트로 **명확하게 읽을 수 있다.** 체크 아이콘(softGreen)이 상단에 배치되어 안심 느낌이 강화되었다. 레이더 차트가 6개 축 레이블을 모두 잘림 없이 표시한다. 영역별 결과 카드("몸 움직임에서 아기가 힘들어할 수 있어요")가 깔끔하게 정렬되어 있다. |
| Originality | 7 | 7 | 6각 레이더 차트의 시각화가 독창적이다. "이렇게 해보세요" 섹션이 맥락적 행동 가이드를 제공한다. 이 부분은 변경 없음. |
| Craft | 4 | 7 | **가장 극적으로 개선된 영역.** 이전에 "치명적"이었던 안심 메시지 텍스트가 완전히 해결. 라이트 모드: mintTint(#F0F9F0) 배경 + darkBrown(#3D3027) 텍스트 = WCAG AA 기준 충족. 다크 모드: darkCard 배경 + darkTextPrimary + softGreen border (코드 확인). 레이더 차트: 컨테이너 200dp -> 260dp, 레이블 SizedBox 60px -> 80px, 폰트 11sp -> 10sp, overflow visible. 모든 레이블("몸 움직임", "감각 반응", "집중과 관심", "소리와 표현", "마음과 관계", "생활 리듬")이 잘림 없이 표시됨을 스크린샷으로 확인. |
| Functionality | 6 | 7 | 안심 메시지가 읽을 수 있게 되어 핵심 기능(안심 전달)이 복원되었다. 레이더 차트 레이블이 완전히 보여 정보 전달이 완전하다. 체크리스트 -> 위험 신호 -> 결과 플로우가 정상 작동. |
| **평균** | **5.75** | **7.25** | **PASS** |

### 개선된 점
- **[P0 해결] 안심 메시지 텍스트 대비:** mintTint 배경 위 darkBrown 텍스트로 명확한 가독성 확보. 다크 모드에서는 darkCard + darkTextPrimary로 전환 (코드 확인).
- **[P0 해결] 레이더 차트 레이블 잘림:** 컨테이너 260dp, 레이블 80px, 10sp 폰트, overflow visible. "마음과 관계", "집중과 관심" 등 5자 레이블 완전 표시 확인.
- **다크 모드 border:** 차트 카드, 결과 카드 등에 isDark 조건부 border 추가.

### 잔여 이슈
- **데이터 없는 상태의 레이더 차트:** 모든 점수가 0일 때 차트 중심에 작은 점만 표시된다. "아직 체크한 항목이 없어요" 같은 오버레이가 있으면 더 좋지만, 0점으로 표시되는 것도 기능적으로 잘못된 것은 아니다. 경미한 개선 기회.

## 4. 마이 탭

### 이전 점수: 5.75 (FAIL)
### 이전 핵심 이슈: 템플릿 느낌 강함, 디자인 차별화 부족, 아이콘 없음, 구분선 없음

### 스크린샷 평가

| 기준 | 이전 | 현재 | 코멘트 |
|---|---|---|---|
| Design Quality | 6 | 8 | **가장 드라마틱한 개선.** warmOrange 그라데이션 상단 악센트 바가 프로필 카드에 브랜드 정체성을 부여한다. warmOrange 아이콘 + 텍스트 섹션 헤더("알림 설정", "화면 설정")가 시각적 위계를 강화한다. 각 설정 그룹이 rounded card로 묶여 있어 그룹핑이 명확하다. |
| Originality | 4 | 7 | 이전의 "시스템 설정 앱" 느낌에서 벗어나, 앱 고유의 디자인 언어가 적용되었다. 프로필 카드의 warmOrange 악센트 바 + 큰 아바타(28dp radius) + warmOrange 링이 프리미엄 감성을 준다. "+ 아기 추가하기" 버튼이 warmOrange outline + add_circle 아이콘으로 브랜드 일관성이 있다. 각 설정 항목 앞 leading 아이콘(wb_sunny, edit_note, auto_awesome, celebration, dark_mode, text_increase, volume_off 등)이 기능 식별을 돕는다. |
| Craft | 6 | 8 | 섹션 간 간격이 균일하다(sectionGap). 카드 내 항목 사이에 indent된 Divider가 적용되어 항목 구분이 명확하다. 토글 스위치의 active(warmOrange)/inactive(mutedBeige) 색상이 일관적이다. "수정 ->" 버튼에 warmOrange 10% 배경이 추가되어 터치 가능 영역이 시각적으로 강조되었다. leading 아이콘의 크기(22dp)와 색상(warmGray)이 일관적이다. |
| Functionality | 7 | 7 | 설정 항목이 명확하게 레이블링되어 있다. "데이터" 섹션(내보내기, 백업)과 "정보" 섹션(앱 정보, 약관, 개인정보, 콘텐츠 출처)이 추가 구조화되어 있다. **다크 모드 토글이 여전히 비작동** -- 탭해도 라이트 모드로 전환되지 않는다. 이 부분은 기능 버그로, 이전 리뷰에서도 동일하게 보고된 pre-existing 이슈. |
| **평균** | **5.75** | **7.5** | **PASS** |

### 개선된 점
- **카드 그룹핑:** 각 설정 섹션이 rounded card로 감싸져 시각적 그룹 형성.
- **Leading 아이콘 추가:** 모든 설정 항목에 의미 있는 아이콘 추가 (13개 아이콘).
- **섹션 헤더 강화:** warmOrange 아이콘 + 굵은 텍스트 헤더로 섹션 구분.
- **Divider 추가:** 카드 내 항목 사이에 indent(xxl + sm) 적용된 구분선.
- **프로필 카드 강화:** warmOrange gradient 악센트 바 + 큰 아바타 + warmOrange 링 + 수정 버튼 배경.
- **"+ 아기 추가하기" 브랜드 통일:** warmOrange outline border + add_circle 아이콘.

### 잔여 이슈
- **다크 모드 토글 비작동:** pre-existing 기능 버그. `settings.setThemeMode()` 호출은 확인되나, 앱 전체의 ThemeMode 반영 로직에 문제가 있는 것으로 추정. fix-result에서 "별도 기능 버그 조사 필요"로 명시.
- **프로필 아바타:** 시스템 이모지(child_care) 기반. 커스텀 일러스트 미적용.

## 판정: PARTIAL

**근거:**
- 이전 FAIL 4개 화면 중 3개가 PASS로 전환되었다 (발달 체크 +1.5, 마이 +1.75, 홈 +0.5).

## 기록 탭 FAIL 피드백 (Generator용)

기록 탭은 6.75점으로 PASS 기준(7.0)에 0.25점 미달한다. 핵심 이슈는 해결되었으나, 다음 개선으로 경계선을 넘을 수 있다:

### FAIL-1: 비활성 카운터 버튼(-) 시각적 존재감 부족
- **위치:** `lib/features/record/widgets/counter_card.dart:218-223`
- **현재:** isDisabled일 때 muted 30% 배경 + muted border로, 버튼이 거의 투명해 보인다. 빈 공간으로 오인될 수 있다.
- **수정 제안:** 비활성 상태에서도 최소한의 존재감을 유지하도록 muted 배경을 40-50%로, border를 50%로 올리거나, 회색 아이콘을 유지하되 border만이라도 선명하게 할 것. 버튼이 "있다"는 것이 인식되어야 한다.

### FAIL-2 (경미): 카운터 카드 간 시각적 단조로움
- **현재:** 수유/배변 카드가 동일한 구조(emoji + label + counter)로 반복되어 시각적 흥미가 부족하다.
- **수정 제안:** 각 카드 좌측 상단에 emoji를 더 크게 배치하거나, 카드 배경에 미세한 gradient를 적용하면 시각적 변화를 줄 수 있다. 그러나 이것은 부가적 개선이지 필수가 아니다.
