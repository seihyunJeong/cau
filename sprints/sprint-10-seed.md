# Sprint 10: 콘텐츠 시드 데이터 (2-3주차 추가 + 0-1주차 보완)

## 1. 구현 범위

### 대상 기능

- **2-3주차 활동 시드 데이터** (5개 활동) -- PDF 원고 제2장 "연결이 늘어나는 시간" 기반
  - 활동 1: 깨어남 신호 릴레이 (감각 전환 / 시각-청각-촉각 연결)
  - 활동 2: 미니 시선 산책 (시각 / 짧은 시선 이동)
  - 활동 3: 어깨-등-골반 파도안기 (안기 / 신체 지지 경험)
  - 활동 4: 가슴언덕 탐험 (자세 / 짧은 엎드림 대체)
  - 활동 5: 발바닥 문답놀이 (촉각 / 하지 반응 관찰)
- **2-3주차 발달 체크리스트 시드 데이터** (18문항 -- 6영역 x 3문항)
  - PDF p.67~69 "2-4. 부모 관찰 기록용 체크리스트" 기반
  - 0-1주차보다 발달된 기준으로 작성 (깨어남 시간 증가, 짧은 상호작용 반응 등)
- **2-3주차 위험 신호 시드 데이터** (6개 항목)
  - PDF p.71 기반, 0-1주차와 동일한 6개 항목 (두 주차 공통 위험 신호)
- **2-3주차 WeekContent** 보완 (기존 weekIndex=1 데이터 검증 및 누락 필드 보강)
- **2-3주차 관찰 기록 항목** -- PDF p.60~64 "부록 2. 2-3주 활동 후 자연 반응 관찰표" 기반
  - 1단계: 활동 직후 전체 상태 관찰 (8항목 -- 0-1주차 7항목에서 확장)
  - 2단계: 자연 반응 관찰 (8항목 -- 0-1주차 7항목에서 시각적 주의/청각-사회적/자세 후 안정/하지 반응 추가, 보행반사 제외)
- **2-3주차 연결 반사 매핑 데이터** -- reflex_data.dart에 2-3주차 활동의 linkedReflex 추가
- **기존 0-1주차 데이터 검증 및 보완**

### 관련 문서 참조

- 서비스기획서: 섹션 2-2 (현재 콘텐츠 범위), 섹션 2-3 (6개 발달 관찰 영역), 섹션 3-3 (주차별 활동 가이드)

### 사용 디자인 컴포넌트

이 스프린트는 순수 데이터 입력 스프린트로, 새로운 UI 컴포넌트를 생성하지 않는다.

## 5. observation_items_seed.dart 함수 시그니처 변경

현재 `getObservationStep1Items()`, `getObservationStep2Items()`는 주차와 무관하게 단일 목록을 반환한다.
2-3주차는 항목이 다르므로, 주차별 분기가 필요하다.

### 변경 방안

```dart
// 기존: List<ObservationItem> getObservationStep1Items()
// 변경: List<ObservationItem> getObservationStep1Items({int weekIndex = 0})
```

weekIndex 파라미터를 추가하되, 기본값 0을 유지하여 기존 호출부의 하위 호환성을 보장한다.
내부적으로는 Map<int, List<ObservationItem>> 구조로 주차별 데이터를 관리한다.

**호출부 수정 필요 여부:** 기존 호출부는 기본값 0으로 동작하므로 수정 불필요. 다만 관찰 기록 화면에서 현재 주차를 전달하도록 수정이 권장된다 (이 수정은 observation_items_seed.dart 내에서 fallback 처리하여 안전하게 대응).
